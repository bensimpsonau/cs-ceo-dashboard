Sunday Batch Filming Workflow

Ben films all 21 reels for the week in a single Sunday session.

  1. 8:00am - ClawBot delivers 21 reel briefs (hook, script outline, ManyChat keyword, pillar tag) to ClickUp
  2. 9:00am - Ben reviews briefs, reorders, trims. Marks any cuts.
  3. 10:00am - 1:00pm - Ben films all 21 reels (approx 8 min per reel including setup)
  4. 1:30pm - Raw footage uploaded to shared Google Drive folder
  5. 2:00pm - Georgio (editor) picks up footage. Target: all 21 reels edited by Wednesday EOD.
  6. As edited - Each reel moves to "In Review" in ClickUp. Ben batch-approves in 30 min blocks.
  7. Post-approval - Approved reels queued with scheduled post times for the week

Comment & Feedback Guidelines

  • Be specific: "Change the hook to lead with the number" not "make it better"
  • Use ClickUp comments: All feedback lives on the task. Not Slack, not text messages.
  • Tag the person: @mention the person who needs to action the feedback
  • One round max: If it needs more than one round of revisions, jump on a 5-minute call
  • Time-box: Reviews must be completed within 4 hours of assignment (business hours)

Version Control

  • Every revision gets uploaded as a new attachment on the task (v1, v2, v3)
  • The approved version is pinned as the top comment with "FINAL" label
  • Never overwrite previous versions. Keep the trail.
  • Google Drive links in the task always point to the latest version

Ben's Final Approval Checklist

Every piece of content must pass all five checks:

  • Voice Check: Does this sound like me / CS? Would I say this in a conversation?
  • Brand Check: No "crypto" (use "digital assets"). No em dashes. No hype language. No urgency tactics.
  • CTA Check: Correct ManyChat keyword? Link goes to collectiveshift.co/cryptoblueprint?
  • Accuracy Check: Any claims backed by data? 116% CAGR noted as "verified and audited by Crypto Tax Australia"?
  • Platform Check: Right format, dimensions, and character count for the target platform?

Emergency Publishing Protocol

Market-moving events (Bitcoin crashes/surges, regulatory news, major hacks)
  1. Ben decides to publish (no one else triggers emergency protocol)
  2. ClawBot drafts a take within 15 minutes, sent to Ben for review
  3. Ben edits and approves in one pass (skip normal review flow)
  4. Post to X/Twitter first (fastest reach), then Instagram Story, then reel if warranted
  5. Status in ClickUp: create task, set status directly to "Published", tag as "Emergency"
  6. EDM can be triggered same-day if Ben decides the event warrants it
  7. Rule: Emergency content must still pass voice and brand checks. Speed does not override quality.

ClawBot in the Review Process

Before any content reaches a human reviewer, ClawBot runs a brand compliance scan.
  • Checks for banned words: "crypto" (should be "digital assets"), em dashes, "to the moon", "WAGMI", "not financial advice" (CS never uses this disclaimer), hype language
  • Checks CTA: ManyChat keyword matches the content pillar
  • Checks voice: Compares against Ben's Voice Profile for tone and style markers
  • Flags issues as comments on the ClickUp task before it hits Ben's queue
  • This saves Ben 2-3 minutes per piece (across 61 pieces/week = ~3 hours saved)

05 Content Calendar Integration

Mapping the 90-Day Calendar into ClickUp

The Evergreen Content System contains 103+ content concepts organized by pillar and format. Here is how to bring that into ClickUp:

  1. Create "90-Day Content Calendar" list in the Content Strategy space
  2. Import all 103 concepts as tasks with Content Pillar, Content Type, and Target Platform fields pre-filled
  3. Assign to weeks: Spread across 13 weeks (90 days). Use the "Week Number" custom field.
  4. Each Monday standup: Pull that week's concepts from the 90-day list and create production tasks in the appropriate Content Production lists
  5. Recycle: At week 14, the cycle restarts. Evergreen content gets refreshed with updated data and re-queued.

Conflict Resolution

When two pieces compete for the same time slot:

  1. Time-sensitive wins. Market events, product launches, and limited-time content always take priority.
  2. Higher-pillar wins. BUILD (40%) content takes priority over YIELD (10%) content.
  3. The bumped piece moves to the next available slot on the same platform, same day if possible.
  4. Never stack. Do not post two pieces on the same platform within 2 hours of each other (algorithm penalty).
  5. Ben breaks ties. If it is not obvious, Ben decides. This should happen less than once per week.

Ben's Voice Checklist

For anyone writing in Ben's voice (including ClawBot):

✓ Do This

  • Write like you talk. Short sentences.
  • Use "digital assets" not "crypto"
  • Lead with conviction, not hype
  • Use specific numbers (116% CAGR, $80 ETH entry)
  • Sound like an educator who invests his own money
  • Use "you" and "I" freely
  • Reference frameworks (Shift OS, Blueprint)
  • Keep paragraphs to 2-3 sentences max

✗ Not That

  • No em dashes anywhere. Ever.
  • No "crypto" (always "digital assets")
  • No hype: "massive gains", "to the moon", "generational opportunity"
  • No urgency tactics: "only 3 spots left", "price going up"
  • No "not financial advice" disclaimer
  • No corporate speak: "leverage", "synergize", "utilize"
  • No passive voice where active works

Do This / Not That

✓ Do This

  • Update ClickUp status at every stage change
  • All feedback as ClickUp comments (not Slack DMs)
  • Use templates for every new task
  • Check the approval queue twice daily
  • Film in batches (Sunday for reels, weekly for YouTube)
  • Let ClawBot draft first, then add your voice
  • Track performance 48 hours after publish
  • Plan 2 weeks ahead, execute 1 week at a time

✗ Not That

  • Request content via Slack messages or verbal asks
  • Skip the review stage "because it is urgent"
  • Leave tasks in Draft for more than 48 hours
  • Approve your own content (even Ben gets ClawBot scan first)
  • Post without checking ManyChat keyword is set
  • Use scheduling tools before the team can hit 61/week manually
  • Add new platforms before mastering current ones
  • Overwrite previous asset versions (keep v1, v2, v3)

Emergency Content Protocol

For market crashes, regulatory news, major hacks, or breaking events:

  1. Ben triggers. No one else decides to go to emergency mode.
  2. ClawBot drafts a take within 15 minutes.
  3. Ben edits + approves in a single pass (5-10 minutes).
  4. Post order: X/Twitter (fastest) then Instagram Story then Reel (if warranted).
  5. ClickUp: Create task, status = Published, tag = Emergency.
  6. Optional: Same-day EDM if Ben decides the event warrants it.
  7. Rule: Speed does not override quality. Voice and brand checks still apply.
